umbra templates run on [minijinja 2](https://docs.rs/minijinja/latest/minijinja/), which ships a Jinja2-compatible filter set out of the box. umbra does not yet register any custom filters of its own — the full set you have available today is the minijinja standard library. Custom filter registration through `Plugin::on_ready` is [deferred](rendering-html#what-ships-at-v1-vs-deferred) until the admin plugin lands at M11.

This page catalogues the filters and globals umbra users reach for most, with examples written in the umbra template style.

## String filters

| Filter | Purpose | Usage |
|--------|---------|-------|
| `upper` | Convert to uppercase | `{{ name | upper }}` |
| `lower` | Convert to lowercase | `{{ name | lower }}` |
| `title` | Title-case each word | `{{ name | title }}` |
| `capitalize` | Uppercase first character, lowercase the rest | `{{ name | capitalize }}` |
| `trim` | Strip leading and trailing whitespace | `{{ text | trim }}` |
| `replace(from, to)` | Replace all occurrences of `from` with `to` | `{{ s | replace("old", "new") }}` |
| `truncate(length)` | Shorten to at most `length` characters, appending `...` | `{{ bio | truncate(120) }}` |
| `wordcount` | Count whitespace-delimited words | `{{ body | wordcount }}` |
| `striptags` | Remove HTML tags from a string | `{{ html | striptags }}` |
| `safe` | Mark a value as HTML-safe, bypassing autoescape | `{{ content | safe }}` |
| `escape` | HTML-escape the value (same as the autoescaper, but explicit) | `{{ text | escape }}` |
| `urlencode` | Percent-encode a string for use in a URL query parameter | `{{ query | urlencode }}` |
| `indent(width)` | Indent each line of a multiline string by `width` spaces | `{{ code | indent(4) }}` |

```html
{# templates/profile.html #}
<h1>{{ user.name | title }}</h1>
<p>{{ bio | truncate(120) }}</p>
<p>{{ description | striptags | truncate(200) }}</p>
```

## Number filters

| Filter | Purpose | Usage |
|--------|---------|-------|
| `abs` | Absolute value | `{{ value | abs }}` |
| `round(precision)` | Round to `precision` decimal places (default 0) | `{{ price | round(2) }}` |
| `float` | Cast to float | `{{ value | float }}` |
| `int` | Cast to integer | `{{ value | int }}` |

```html
<p>Price: {{ product.price | round(2) }}</p>
<p>In stock: {{ inventory | abs }}</p>
```

## List and collection filters

| Filter | Purpose | Usage |
|--------|---------|-------|
| `length` | Number of items in a sequence or characters in a string | `{{ items | length }}` |
| `first` | First element of a sequence | `{{ items | first }}` |
| `last` | Last element of a sequence | `{{ items | last }}` |
| `reverse` | Reverse a sequence | `{{ items | reverse }}` |
| `sort` | Sort a sequence (ascending by default) | `{{ items | sort }}` |
| `unique` | Remove duplicate values from a sequence | `{{ items | unique }}` |
| `join(sep)` | Join items with a separator string (default `""`) | `{{ tags | join(", ") }}` |
| `list` | Coerce a value to a list | `{{ value | list }}` |
| `min` | Smallest value in a sequence | `{{ scores | min }}` |
| `max` | Largest value in a sequence | `{{ scores | max }}` |
| `sum` | Sum of numeric values in a sequence | `{{ prices | sum }}` |
| `map(attr)` | Extract an attribute from every item in a sequence | `{{ items | map(attribute="name") }}` |
| `select(attr)` | Filter a sequence to items where `attr` is truthy | `{{ items | select("active") }}` |
| `reject(attr)` | Filter a sequence to items where `attr` is falsy | `{{ items | reject("archived") }}` |
| `batch(size)` | Group items into chunks of `size` | `{{ items | batch(3) }}` |
| `slice(count)` | Slice a sequence into `count` parts | `{{ items | slice(3) }}` |

```html
{# Comma-separated tag names #}
<p>Tags: {{ article.tags | map(attribute="name") | join(", ") }}</p>

{# First three items only #}
{% for item in items | list | slice(3) | first %}
  <li>{{ item.title }}</li>
{% endfor %}

{# Total pages: 12 #}
<p>Total pages: {{ pages | length }}</p>
```

## Default and type-testing filters

| Filter | Purpose | Usage |
|--------|---------|-------|
| `default(value)` | Return `value` if the variable is undefined or falsy | `{{ bio | default("No bio provided.") }}` |
| `d(value)` | Alias for `default` | `{{ bio | d("No bio provided.") }}` |
| `dictsort` | Sort a dict by its keys | `{{ mapping | dictsort }}` |
| `items` | Return `(key, value)` pairs from a dict | `{{ mapping | items }}` |
| `tojson` | Serialize the value to a JSON string | `{{ payload | tojson }}` |

```html
<p>{{ user.bio | default("No bio provided.") }}</p>
<script>
  const data = {{ payload | tojson }};
</script>
```

## Control-flow tags (not filters, but commonly confused)

These are Jinja2 tags, not pipe filters. They're included here because they solve the same problems humanize helpers solve in Django.

```html
{# Conditional blocks #}
{% if articles | length == 0 %}
  <p>No articles yet.</p>
{% else %}
  <p>{{ articles | length }} article{% if articles | length != 1 %}s{% endif %}</p>
{% endif %}

{# Loop helpers inside a for block #}
{% for item in items %}
  {# loop.index is 1-based; loop.index0 is 0-based #}
  <li class="{% if loop.first %}first{% endif %} {% if loop.last %}last{% endif %}">
    {{ loop.index }}. {{ item.name }}
  </li>
{% endfor %}
```

## Global functions

minijinja exposes a small set of global functions callable anywhere in a template expression.

| Function | Purpose | Usage |
|----------|---------|-------|
| `range(start, stop, step)` | Generate an integer range (like Python's `range`) | `{% for i in range(5) %}` |
| `dict(...)` | Construct a dictionary inline | `{% set d = dict(key="value") %}` |
| `namespace(...)` | A mutable namespace for storing values across loop iterations | `{% set ns = namespace(count=0) %}` |
| `lipsum(n)` | Generate `n` paragraphs of Lorem Ipsum (useful in layout prototyping) | `{{ lipsum(2) }}` |

```html
{% for i in range(1, 6) %}
  <p>Item {{ i }}</p>
{% endfor %}
```

## The `safe` filter and autoescape

umbra enables autoescape on every `.html` and `.htm` template. A value that contains `<script>` will be emitted as `&lt;script&gt;` automatically. To render pre-trusted HTML — for example, a sanitised rich-text field — pipe it through `safe`:

```html
{# Only do this for content you have already sanitised server-side #}
<div class="body">{{ article.rendered_html | safe }}</div>
```

There is no global escape bypass. See [Rendering HTML](rendering-html#autoescape-is-on-by-default) for the security rationale.

## What is missing today: humanize helpers

Django's `django.contrib.humanize` provides filters like `naturalday`, `naturaltime`, `intcomma`, `ordinal`, and `filesizeformat`. minijinja 2 does not ship these, and umbra has not yet added them as custom filters. The deferred design (in `rendering-html.mdx` and `arch.md §4.5`) shows that custom filters will come through `Plugin::on_ready` once the engine's ambient pattern supports mutable registration. When humanize filters land they will be documented here.

In the meantime, compute humanized representations in the handler and pass them as context values:

```rust
// handler: pre-compute the human-readable string
let context = context! {
    article,
    created_label => humantime::format_duration(elapsed).to_string(),
};
let html = umbra::templates::render("article.html", &context)?;
```

```html
{# template: consume it like any other variable #}
<time>{{ created_label }}</time>
```
