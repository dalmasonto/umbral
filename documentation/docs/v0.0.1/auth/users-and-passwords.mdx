`umbra-auth` ships the canonical User model, argon2 password hashing, and
the `create_user` / `authenticate` / `set_password` helpers. It's the first
crate under `plugins/` and the structural proof of the [Plugin
contract](/v0.0.1/plugins/the-plugin-trait).

## Register the plugin

```rust
use umbra::prelude::*;
use umbra_auth::AuthPlugin;

App::builder()
    .settings(settings)
    .database("default", pool)
    .plugin(AuthPlugin)
    .build()?;
```

`AuthPlugin` contributes one model (`AuthUser`) under the `"auth"` plugin
namespace. Run `makemigrations` and `migrate` to create the `auth_user`
table.

## The helpers

```rust
use umbra_auth::{authenticate, create_user, set_password};

// Create
let mut alice = create_user("alice", "alice@example.com", "hunter2").await?;

// Authenticate
let user = authenticate("alice", "hunter2").await?;
// or: AuthError::InvalidCredentials on wrong-password OR unknown-username
// (the variant is the same so callers can't enumerate accounts)

// Rotate
set_password(&mut alice, "stronger-password").await?;
```

`hash_password` and `verify_password` are exposed too, for hand-built flows
that don't want the helpers to touch the database.

<Callout type="info">
Passwords are hashed with argon2 using the crate's default parameters. The
PHC-encoded hash is self-describing, so future parameter upgrades can verify
old hashes and silently re-hash on next login.
</Callout>

## The AuthUser shape

```rust
pub struct AuthUser {
    pub id: i64,
    pub username: String,        // UNIQUE
    pub email: String,
    pub password_hash: String,
    pub is_active: bool,
    pub is_staff: bool,
    pub is_superuser: bool,
    pub date_joined: DateTime<Utc>,
    pub last_login: Option<DateTime<Utc>>,
}
```

`authenticate` filters on `is_active = 1`, so disabled accounts reject
unconditionally.

## What ships at M9 v1 vs deferred

Deferred to later milestones (per `docs/specs/outlines/auth-and-sessions.md`):

- Custom user model swap via a `UserProvider` associated type.
- Permissions, groups, the pluggable auth-backend chain.
- The `Auth<U>` request extractor and `#[login_required]` layer. Both
  require `Plugin::middleware()` from the [Plugin trait](/v0.0.1/plugins/
  the-plugin-trait).
- Login / logout / password-reset HTTP flows. Both need `umbra-sessions`
  (a sister plugin) and `umbra-email`, neither shipped yet.
