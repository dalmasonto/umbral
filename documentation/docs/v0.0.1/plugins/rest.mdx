`umbra-rest` walks the model registry and mounts a JSON CRUD surface for every model you've registered. Add the plugin, get `/api/<table>/` on List + Retrieve + Create + Update + Delete, plus filtering, ordering, search, pagination, and an OpenAPI spec everything else (the playground UI, code generators, third-party clients) reads from.

```rust
use umbra::prelude::*;

App::builder()
    .model::<Article>()
    .plugin(umbra_rest::RestPlugin::default())
    .build()?;
// → GET/POST/PATCH/PUT/DELETE under /api/article/
```

The full surface lives in the [**REST**](/docs/v0.0.1/rest) section:

- [**Overview**](/docs/v0.0.1/rest) - what gets generated, customising responses, custom actions
- [**Authentication**](/docs/v0.0.1/rest/authentication) - `SessionAuthentication`, `BearerAuthentication`, `ChainAuthentication`, writing your own
- [**Permissions**](/docs/v0.0.1/rest/permissions) - `AllowAny`, `IsAuthenticated`, `IsStaff`, `ReadOnly`, `Or` / `And`, writing your own

The plugin contract itself is documented in `arch.md`; the `rest/` folder above covers the everyday consumer surface.
