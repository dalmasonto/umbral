# shop

A full e-commerce demo generated by `umbra startproject shop`.

## What's in the project

| File | What it shows |
|---|---|
| `src/main.rs` | App wiring: models, plugins, routes, auto-migrate, seed data |
| `plugins/content/` | Content plugin: blog, pages, FAQ, navigation, media, SEO |
| `plugins/ecommerce/` | E-commerce plugin: products, orders, customers, coupons, reviews |
| `Product` model | `ForeignKey<Category>`, choices enums, ORM QuerySet, `#[derive(Model)]` |
| `/` route | Featured products + brands, template rendering with context |
| `/products` | Product catalog listing |
| `/products/{id}` | Product detail page |
| `/dashboard` | `login_required_html("/login")` layer, `LoggedIn<AuthUser>` extractor |
| `/staff-only` | `permission_required_html("ecommerce.view_product", "/login")` — 403 for users without the perm |
| `/api/product/` | REST CRUD with query-string filtering |
| `/api/order/` | Order REST endpoint |
| `/admin/` | Auto CRUD UI for all models |
| `/api/docs/` | Swagger UI from OpenApiPlugin |

## Running

```bash
# First run — applies migrations, seeds demo data, and starts the server:
cargo run -- serve

# Separate steps (production pattern):
cargo run -- migrate
cargo run -- serve

# Create a superuser to log in to the admin:
cargo run -- createsuperuser

# Explore the scaffold:
cargo run -- showmigrations
cargo run -- makemigrations
```

## Permissions

The [`umbra-permissions`](../../plugins/umbra-permissions) plugin is active. On boot it auto-creates ContentTypes and the four standard permissions (`add`, `change`, `delete`, `view`) for every registered model. The `/staff-only` route demonstrates `permission_required_html` gating — superusers bypass the check; regular users need the explicit permission (or group membership) to access it.
