- ✅ **Postgres rollout Phase 2.5 — backend-agnostic QuerySet.** The `Model` trait dropped its `for<'r> FromRow<'r, SqliteRow>` supertrait — it's now metadata-only, with the FromRow bound carried on each QuerySet terminal (the conjunction of both backends' impls). `#[derive(sqlx::FromRow)]` already emits a generic-over-`R: Row` impl, so user structs satisfy both bounds via one derive with no per-backend ceremony. QuerySet's `explicit_pool` slot is now `Option<DbPool>`; `.on(&SqlitePool)` keeps its existing shape and a new `.on_pg(&PgPool)` registers a Postgres pool. Each terminal (`fetch`/`first`/`count`/`exists`) matches on the resolved `DbPool` variant: SQLite path uses `SqliteQueryBuilder` + a SqlitePool executor; Postgres path uses `PostgresQueryBuilder` + a PgPool executor. Required enabling `sqlx-postgres` on sea-query-binder. **Net effect**: the same `Manager<T>` works against both pool variants — `Post::objects().filter(...).fetch()` is now backend-agnostic, matching a clean ORM shape. 270 tests pass (Phase 2 was 268; +2 compile-only type-level pins). One `#[ignore]`'d round-trip test runs against a live PG when `UMBRAL_TEST_POSTGRES_URL` is set. User docs at `documentation/docs/v0.0.1/backends/postgres.mdx`. Commit: `6f24fbd` (+ docs `90b7f87`).

## Postgres rollout — complete

Every Phase 4 sub-phase has shipped. Phases 1 + 2 + 2.5 + 3 + 4.0 + 4.1 + 4.2.1 + 4.2 + 4.4 + 4.2.2 + 4.3 + 4.5 cover the full Postgres surface umbral targeted:

- DbPool enum + backend-aware connect + boot-time mismatch detection.
- Backend-aware DDL rendering + per-variant migrate apply loop.
- Backend-agnostic QuerySet terminals (PG-only models use `.fetch_pg(&pool)` etc.).
- Postgres inspectdb via `information_schema`.
- Portable Json field with cross-backend operators (Postgres `->`/`->>`/`?` plus SQLite `json_extract` fallback).
- Postgres-only field types with `field.backend` boot-time gating: Array, Inet, Cidr, MacAddr, FullText.
- Postgres-specific operators: array `@>`/`<@`/`&&`, full-text `@@ to_tsquery` / `@@ websearch_to_tsquery`.
- Row-Level Security plugin (`umbral-rls`) with policy declaration + idempotent application at boot.

Anything missing is now a normal-priority enhancement rather than a Phase 4 gap — see the per-feature rustdoc / FEATURES.md commit messages for the documented deferrals (Cidr opt-in attribute, JSON containment operators, array-element-typed columns, etc.).

Each piece is its own session: too much for autonomous ship-and-verify in one round, and a half-finished refactor with broken tests is worse than no refactor.
