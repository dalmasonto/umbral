# 0.0.5 — 2026-07-05

The **security-hardening release** — the output of a deep production-readiness audit (`audit_2`). Secure-by-default across the board: default-deny mass assignment, forced row-level security with a per-request GUC (no cross-request leak), hardening headers + body-size/timeout limits, session/CSRF/permission tightening, and closed XSS/enumeration/error-leak holes. Also ships custom admin views, object-level REST row scoping, and recursive N-level nested writes.
