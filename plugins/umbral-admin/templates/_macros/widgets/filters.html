{# filters.html ============ The widget filter strip — one control row rendered for EVERY widget kind. Before this existed, the only filter in the admin was a `[7d][30d][90d]` chip strip hardcoded inside `line.html`, so a bar chart or a table could not be filtered at all and no widget could offer anything but a period. Filters are now declared on the widget (`Widget::filter(..)`) and rendered here, whatever the kind. Every control hx-gets the widget's own data endpoint and swaps the widget body in place. Each one appends `f.carry` — a query fragment holding the OTHER filters' current values, computed in Rust — so picking a status does not silently reset the period. Params: widget_key — the widget's key; the strip is skipped without it. filters — resolved filters from the handler. Each carries `key`, `label`, `kind`, its options, the `active` value, `active_start` / `active_end` for a range, and `carry`. #} {% macro filter_strip(widget_key, filters) %} {% if widget_key and filters %}
