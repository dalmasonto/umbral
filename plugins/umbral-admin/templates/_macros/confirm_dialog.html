{# JS-driven delete: fetch DELETE with the CSRF token, check the real HTTP status, refresh the table, and toast honestly. A bare `hx-delete` form would (a) need the token threaded and (b) swap the handler's empty body into #table-body, blanking it. `umbral._deleteRow` lives in the changelist JS (data_table.html) — the only place this dialog opens. The URL rides a `data-delete-url` attribute rather than being spliced into the `onclick` JS string: an interpolated `_deleteRow('{{ url }}')` is unsafe because HTML-entity escaping does NOT stop JS-string breakout in an event-handler attribute (the browser decodes entities before the JS parser runs).
